To build Boost 1.84 in Git Bash, clone the repository, check out the boost-1.84.0 tag, and initialize submodules. 
Use ./bootstrap.sh followed by ./b2 to compile. On Windows, ensure a compiler (like MinGW or MSVC) is installed and accessible in the path. 

Step-by-Step Instructions
Clone the Repository
Run this command in Git Bash to clone the full repository with submodules:
bash
git clone --recursive https://github.com/boostorg/boost.git
cd boost
Checkout Version 1.84
Switch to the specific tag for version 1.84.0:
bash
git checkout boost-1.84.0
git submodule update --init --recursive
Bootstrap the Build System
Run the bootstrap script to prepare the build engine (b2):
bash
./bootstrap.sh
Note: If you are on Windows and using MinGW, you might need to use cmd /c bootstrap.bat mingw instead.
Build Boost
Run b2 to build the libraries. You can specify a prefix for the installation directory:
bash
./b2 install --prefix=../boost-1.84-install
Optional: Use -j4 to use 4 cores for faster compilation. 

Key Notes
Header-only: Most Boost libraries are header-only and do not require compilation, but libraries like filesystem or thread need this build process.
Alternative: You can download the source tarball directly from the Boost Website instead of using git.
